# Kiro 语言偏好设置

## 基本语言规则

### 输出语言
- **始终使用中文**进行所有输出和交流
- 所有用户界面文本使用中文
- 错误信息和解释使用中文
- 帮助文档和说明使用中文

### 代码规范
- **变量名和函数名**：使用英文（遵循编程规范）
- **代码注释**：使用中文编写
- **类名和接口名**：使用英文
- **常量名**：使用英文大写，但注释用中文

## 文档编写规范

### Markdown文档
- 所有标题、段落使用中文
- 列表项使用中文
- 代码块中的注释使用中文
- 表格内容使用中文

### 技术文档
- API文档使用中文编写
- 设计文档使用中文
- 需求分析使用中文
- 任务说明使用中文

## 交互规范

### 对话交流
- 优先使用中文回复用户
- 技术术语保留英文原名，但提供中文解释
- 示例和演示使用中文说明
- 反馈和建议使用中文表达

### 专业术语处理
```
英文术语 - 中文解释
CSS - 层叠样式表
API - 应用程序编程接口
UI - 用户界面
UX - 用户体验
```

## 错误处理和调试

### 错误信息格式
```
错误类型：具体错误描述
- 发生位置：文件路径和行号
- 可能原因：分析错误原因
- 解决建议：提供解决方案
```

### 调试输出
- 调试信息使用中文
- 日志记录使用中文
- 状态报告使用中文

## 项目特定规范

### Tokyo Night 主题项目
- 所有与主题相关的说明使用中文
- 颜色变量的注释使用中文
- 样式组件的说明使用中文
- 配置选项的描述使用中文

### 文件命名
- 文件名可以使用英文（便于跨平台）
- 但文件内容和注释使用中文
- README和文档文件使用中文命名

## 质量保证

### 语言一致性检查
- 确保所有输出内容语言一致
- 检查代码注释是否使用中文
- 验证文档是否完全中文化

### 用户体验
- 确保中文表达自然流畅
- 技术解释清晰易懂
- 专业术语使用恰当

## 特殊情况处理

### 中英文混排
- 英文术语与中文之间添加空格
- 示例：`使用 CSS Grid 布局`
- 代码内联时无需空格

### 代码块中的中文
- 字符串字面量可以使用中文
- 但建议使用英文避免编码问题
- 注释必须使用中文
