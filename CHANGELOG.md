# 更新日志

本项目采用语义化版本，遵循 Keep a Changelog 的结构。

## [Unreleased]

### 改进
- 统一主编辑区日期/时间弹层与侧边栏日历的标题着色与文字权重，修正顶部年份/月份/本月项在弹层中的视觉不一致。
- 统一主编辑区日期弹层的周末默认文字颜色为 comment 色阶，并补齐 today/selected/outside 等状态优先级。
- 新增对 `orca-plugin-task-planner` 分段控件（`data-role="mlo-task-views-panel-root"`）的局部样式兼容：保持插件原有交互与布局，仅调和 segmented 背景色并增强亮/暗模式边界清晰度与未选中项文字可读性。

### 修复
- 修复了因将日期/时间面板样式错误扩散到宿主布局层而导致的“弹出面板时主界面整体上移、`header` 被挤压且关闭后不恢复”问题；并修复了因回归时过度约束弹层定位而导致选择弹窗异常的问题，最终改为恢复宿主默认弹层布局并仅调整主编辑区顶部偏移。
- 修复主编辑区小时/分钟选择列表未命中正确 DOM 结构的问题，改为结构型选择器覆盖真实节点。
- 修复小时/分钟选择列表交互：激活项改为仅文字着色、移除背景；移除 Dock 缩放，避免小区域内出现抖动观感与横向滚动风险。

## [1.1.0] - 2026-03-17

### 新增
- 新增主题样式源码目录 `src/theme-css/`，按 `tokens/base/components/feature-overrides/final-overrides` 进行分层维护。
- 新增 `scripts/build-theme-css.cjs` 与 `src/theme-css/manifest.json`，支持按固定顺序拼接生成 `public/tokyo-night.css`。
- 新增 CSS 质量检查：引入 stylelint 最小规则集，提供 `npm run lint:css`。

### 改进
- 构建流程调整为先执行 `build:css` 再进行 TypeScript/Vite 构建，确保发布产物始终由源码分片生成。
- 新增 `npm run check`（`build + lint:css`）作为主题重构后的基础验收命令。
- 文档更新：补充主题 CSS 分层结构、拼接机制与开发检查流程。
- 统一“/ 日期面板”与侧边栏日历的样式与交互逻辑，复用同一套日期/月份/年份视觉规则与动效。
- 块引用箭头回归宿主默认方案，并微调尺寸与垂直位置；hover 时新增弹性回弹动效。

### 修复
- 修复主题卸载时可能误注销同名主题的问题（仅注销本插件注册的主题）。
- 修复“/ 日期选择面板”网格未铺满与暗色模式表现不一致的问题。
- 修复暗色模式下周末日期可读性偏低的问题（默认提升到 comment 色阶）。
- 修复侧边栏标签图标 hover 时覆盖用户自定义颜色的问题（仅在未自定义颜色时应用默认 hover 色）。
- 修复块引用/标签 hover 视觉抖动问题，并调整为仅文字着色不添加背景。
- 修复块引用预览面板在长内容场景下层级线溢出边界的问题。
- 修复 tag 胶囊文字因 hover 命中区扩展导致的偏移问题。

## [1.0.0] - 2026-03-15

### 新增
- Tokyo Night Light / Dark 双模支持：单主题自动随系统或 Orca 亮暗模式切换。
- Excalidraw 白板主题专项样式：覆盖 Island、对话框/菜单、侧栏、工具按钮、库切换器、输入框等关键区域。
- Excalidraw 右键菜单可读性增强：提升暗色背景下 hover 文本对比度。
- 编辑区白板块柔和绿色轮廓：增加低存在感边框与微光，并在 hover 时轻微提亮。

### 改进
- 查询条件框视觉重构：引入细线霓虹边框、多色灯管渐变与亮/暗分档参数。
- 查询条件框背景动效升级：新增多点光源模糊光晕层，并提供 `prefers-reduced-motion` 降级。
- 行内代码 keycap 体系重做：统一亮/暗主题下的顶面、台阶、投影与交互反馈。
- 代码块语法高亮对齐 Tokyo Night 语义：补充 Prism token 映射并改为主题变量驱动（`--tn-code-*`）。
- 修复亮色模式代码符号异常底色：清除 `operator/punctuation/entity` 在代码块中的误命中背景。
- 代码块编辑态高亮补齐：新增 CodeMirror 语法类与压缩类名（如 `ͼp~ͼw`）映射，使“激活编辑态”与“预览态”更一致。
- 代码块行号可读性优化：提升行号颜色对比度，并统一行号 gutter 背景与分隔线，消除白底割裂感。
- 代码块 hover 抖动修复：移除代码块 hover 位移，保留颜色与阴影反馈，降低鼠标移入移出时文本抖动。
- 无序列表圆点着色增强：覆盖编辑态 `orca-block-handle` 及多级列表颜色映射，避免与普通文本块规则冲突。
- 块层级线视觉系统重构：按层级统一默认/hover/active 色阶，并修复激活层级与 Tokyo Night 不一致及联动高亮问题。
- 主题圆角一致性调优：将组件内多处硬编码半径收敛到 `--orca-radius-*` 体系，统一控件与浮层圆角节奏。
- 主编辑区与顶部工具栏边缘关系调优：参考官方主题改为 `margin-top` 落位，修复上边缘截断与层级错觉。
- 行内代码交互修复：缓解相邻行 hover 遮挡，统一 hover/active 下压逻辑并校正“下边缘高亮”视觉偏差。
- 构建流程补强：新增 `postbuild`，自动复制 `public/tokyo-night.css` 到 `dist/tokyo-night.css`。
- 侧边栏与菜单交互优化：统一按钮/菜单 hover 反馈，降低抖动并提升过渡平滑度。
- 亮色模式细节修复：修正新页面按钮对齐、代码块文本清晰度与日历日期 hover 对比表现。
- 主编辑区视觉融合优化：边界与窗体融合、容器定位锚定工具栏、明暗阴影分离调优。
- 侧边栏筛选输入框重构：移除霓虹背景方案，改为单下边框交互，并实现“中间厚两侧薄 -> hover/focus 浸润展开”的线条动画；同时去除该输入框在亮/暗模式下的默认焦点高亮边框。
- 侧边栏标签页指示优化（收藏/标签/页面）：默认无装饰、hover 显示小圆点、激活显示同色短线；短线厚度与位置按视觉反馈微调。
- 日历头部可点击项反馈增强：为年份、月份和“本月”增加 hover 颜色过渡，并在亮/暗模式下分别校正可视度与同色系区分。
- 日历年份/月份选择面板交互优化：复用日期的 Dock 缩放动效；修复右下角触发滚动条问题，限制非必要方向滚动并保留年份页纵向滚动能力。
- 日历切换面板视觉一致性修复：面板及滚动条轨道背景与侧边栏背景对齐，避免出现独立底色块。
- 暗色模式侧边栏图标可读性优化：提升仓库切换、日历左右按钮、筛选框图标等未激活态可见度，并拉开默认态与 hover 态层级差。
- 标签列表图标着色策略修复：避免覆盖用户自定义标签颜色；图标在默认态使用弱化色，仅在对应标签 hover 时与文本同色。

- 采用从左到右的渐变背景（左侧透明 -> 右侧类型色），避免边框超出范围问题
- 右下角 Emoji 图标，悬停时旋转放大并去除灰度

- 移除块展开/收起圆点的 hover 效果，避免与宿主效果冲突
- Callout 块内隐藏块操作圆点，避免与左侧装饰线干涉
- 优化顶部工具栏与编辑区布局，编辑区上边界与工具栏下边界对齐（参考官方主题做法）
- 统一工具栏背景色与窗体控制按钮，消除视觉不一致
- 移除工具栏 mask-image 渐变效果，修复插件按钮背景显示异常问题
- 调整编辑器顶部内边距，改善首行内容的可读性
